# Triage Labels

This repo uses these canonical workflow labels:

| Role | Label |
| --- | --- |
| Needs triage | `needs-triage` |
| Needs reporter input | `needs-info` |
| Ready for an agent | `ready-for-agent` |
| Ready for a human | `ready-for-human` |
| Will not be worked on | `wontfix` |

Additional workflow labels:

| Purpose | Label |
| --- | --- |
| Product requirements document | `prd` |
| Bug report | `bug` |
| Enhancement request | `enhancement` |

Agent-loop control labels:

| Purpose | Label |
| --- | --- |
| Claimed by master loop | `agent-loop:claimed` |
| Active worker task | `agent-loop:in-progress` |
| Blocked task | `agent-loop:blocked` |
| PR ready to merge | `agent-loop:ready-to-merge` |
| PR merged | `agent-loop:merged` |
| Work complete | `agent-loop:done` |
| Automation failed | `agent-loop:failed` |
